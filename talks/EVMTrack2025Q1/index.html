EVM Semantics in HOL4

What do we have so far?

Executable Semantics
  • Phrased as easy-to-read monadic functions in logic
  • Passes all major functionality in the EVM test suite at the Cancun revision
Proofs about the Semantics
  • Gas is strictly consumed, and hence all calls terminate
  • Separation properties: e.g. execution produces the same result in a larger state
  • etc... just getting started with more properties...
Auxiliary formalisation work
  • Recursive Length Prefix coding
  • Merkle Patricia Trees
  • Keccak256 and other hash functions
  • Contract ABI coding... etc...

Project Directions

What are we planning to do?

Verified Compilation for Vyper
  • Formal definition of (a subset of) the Vyper language (funded, in progress)
  • Semantics-preserving compilation from Vyper to EVM (applying for funding)
  • Production-grade verified compilation of Vyper (guiding star)
Proof-Producing Decompilation into Logic
  • Verification technology for arbitrary applications (no matter how produced)
  • Applying for funding to prototype this approach (based on existing machine-code decompilation work)
Separation Logic for Program Verification
  • Some experimental work so far
  • Will design and implement this next
And more...
  • E.g. verified compilation (with CakeML/Pancake) of the EVM (not just to EVM)
  • Free and open source software, all public
  • Welcoming, community-driven development
